Question-1: Difference between map, filter , find , forEach ??

Answer :

map: This method is used to iterate over each element of an array, apply a function to each element, and return a new array with the results.
filter:This method is used to iterate over each element of an array and return a new array containing only the elements that satisfy a certain condition.
find: This method is used to find the first element in an array that satisfies a certain condition, and return that element.
forEach: This method is used to iterate over each element of an array and perform some action on each element. It doesn't return a new array, and it doesn't modify the original array
forEach is used to iterate over an array and perform an action on each element, while map and filter are used to transform an array into a new array with modified or filtered elements. find is used to search for a specific element in an array based on a certain condition.

Question-2: Difference between local storage , session storage , cookies ??

Answer :

JavaScript provides three mechanisms for storing data on the client cookies, session storage, and local storage. Each one has advantages and disadvantages.
Local storage is the most recent mechanism. It allows for larger amounts of data to be stored, but the data is not deleted when the browser is closed. Local storage is useful for storing data that the user will need to access later, such as offline data.
Session storage is similar to cookies, but the data is only stored for the current session. This means that the data will be deleted when the user closes the browser. Session storage is useful for storing data that is sensitive, such as login credentials.
Cookies are the oldest and most well-known mechanism. They are simple to use and well supported by browsers. However, they are limited to 4KB of data and are often used to store data that is not sensitive, such as user preferences.

Question-3: What is eventLoop ? how does it work?

Answer :

The event loop is a mechanism in JavaScript that processes and dispatches events in a loop. It continuously checks for events in the event queue, picks them up, and passes them to the relevant event handler for processing. Asynchronous operations are handled by the browser environment, and when they complete, they're placed in the event queue as new events. The event loop allows JavaScript to handle asynchronous operations efficiently without blocking the main thread.
